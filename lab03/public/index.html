Exercise 3.1


Node.js and Express.js are two very different things. Node.js is an application interpreter/runtime for Javascript, while Express.js is a library that adds additional niceties on top of Node's capabilities, such as the ability to load in "middleware," use templating engines, and do routing, etc.  While Node does include somewhat basic HTTP functionality out of the box, Express provides the necessary packages and functions to be able to easily create a webapp server using Node.


The packages.json file is the file used by Node's package manager, NPM, to determine which dependencies need to be installed in order to run your Node app.  In addition to just listing required packages and their versions, the packages.json file also gives some meta information about the package it is included in, such as the version, author, and a description of what the package does.

Exercise 3.2


Static files in the /static directory are files that Node will simply load and then send to the user without any middleware intervention.  This makes it really easy to have a bunch of Javascript or CSS files that can be easily created and linked to without adding a route for each one.  These files are 'static' in the sense that they are not interpreted by Node or a templating engine before being sent to the user, they are just loaded from disk and sent straight to the user without any other interventions by the server—so, they just don't change.


Well, my server continues to, at /hello, since I changed the route. If the static directive had been placed above the routes section, and an index.html file had been placed into the /static folder, then the static file would have trumped the routed text message.  But, if the route had been placed above the static directive, then the route would have taken precedence over the static files.  In other words, Express will choose the first matching route/file when given a choice.  So, if I had not changed anything, then no, the text would no longer be displayed.
